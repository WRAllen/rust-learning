# 变量

终于可以开始学习变量啦～

首先变量最重要的就是名字，毕竟没有名字怎么知道变量是谁。

## 取名规范

规则：Rust 变量名统一用 snake_case（小写+下划线） 这点和python一致，而go偏向camelCase（例如： helloWorld）

```rust
let user_id = 123;
let cache_hit_rate = 0.98;
let num_tokens = 1024;
```

变量名规范：

1. 以字母或下划线开头

2. 后面可以是：字母 / 数字 / 下划线

## 不叫赋值，叫绑定

这里涉及到rust的核心，他给 `变量 = xx` 的行为不叫赋值，而是叫**绑定(binding)**，下面会具体说明

变量类型总览
| 类型   | 关键字          | 是否可变      | 是否全局 | 常用    |
| ---- | ------------ | --------- | ---- | ----- |
| 普通变量 | `let`        | ❌ 默认不可变   | ❌    | ⭐⭐⭐⭐⭐ |
| 可变变量 | `let mut`    | ✅         | ❌    | ⭐⭐⭐⭐⭐ |
| 常量   | `const`      | ❌         | ✅    | ⭐⭐⭐⭐  |
| 静态变量 | `static`     | ❌（或 mut）  | ✅    | ⭐⭐⭐   |
| 静态可变 | `static mut` | ✅（unsafe） | ✅    | ⭐     |

对于不可变变量，一定把值绑定到变量上，那这个变量和这个值就不能改变了，除非你用同样的变量名，再进行一次绑定操作。

说实话，这里我觉得初学者没必要扣这种细节，当然认知是很重要的这点我赞同，但是对于初学者来说是不友好的

其实，对于初学者来说，你只要记住rust的变量定义分为不可变变量和可变变量
```rust
// 不可变变量
let a = "good";
// 可变变量
let mut b = "rust";
```

## 强类型
还有一点要注意，rust是强类型语言，这点和python这种动态语言不一样（动态语言是运行中时才会知道具体是什么类型），和go一样的强类型

但是话又说回来了，为啥有时候看到rust的变量他没有具体类型呢？其实因为rust的编译器很聪明，很多情况下就和python一样，他知道编译期间

你这个变量的具体类型是啥，只有那些编译器无法推断出的变量，需要你手动声明一下
